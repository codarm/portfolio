Federated learning with inter-dependent edge nodes

Developed Federated- edge frameworks for Urban sensing and interactions accounting for inter-dependent edge nodes and spatio-temporal dependency

OVERVIEW

Computing and analytics are increasingly moving to edge devices (local devices on the field, rather than computation related to them running on servers), to avoid the latency of sending data to cloud servers for processing. The current popular method for edge analytics is to train machine learning models on the cloud and deploy the trained model on the edge device. For new incoming data then, prediction or classification is done on the edge device itself. The model, however, once trained does not get updated based on new data continuously, and therefore the model is not continuously improving, but improving only at a periodic interval (when a model update happens at the server level). Additionally, each edge device will have a specific set of data inputs, that it should learn independently- in some sense a personalized model for that device itself. Once this individual model has been learned, it can be improved by combining it with other edge device models to make it more robust. This does not happen currently in the edge device paradigm in the Urban Sensing context. These shortcomings have led to the development of the federated learning framework, which was initially built to enable mobile phones to collaboratively learn a shared prediction model while keeping all the training data on the device, decoupling the ability to do machine learning from the need to store the data in the cloud. This goes beyond the use of local models that make predictions on mobile devices by bringing model training to the device as well. But one of the major shortcomings of current federated learning models is its ineffectiveness in modeling the complex interdependency between functionally co-related edge devices in real-time decision making (for example Spatio-temporal models), and this remains an open problem.

 

The proposed patent-pending work presents a framework for building machine learning models on Internet of Things edge devices for real-time decision making and prediction which takes into account interdependence between functionally co-related edge devices. Currently, the way decisions are made on edge devices accounts for the input data of that particular device itself. Our framework, which addresses this problem, can be used across use-cases where functionally co-related devices exist. While in a federated learning setup, these nodes currently learn models collaboratively (by averaging weights learned independently at each node after every given interval of time), they do not account for outputs of a given node to influence the prediction of another node in real-time decision-making processes. We provide for this accounting in our framework, through a Bayesian probabilistic dependency of nodes calculated from historical data of outputs of each node. A similar use case would be a cluster of parking lots in an area that can be modeled as interdependent nodes, were filling up 1 parking lot (due to an event near it), would change the predicted parking congestion at other parking lots. Additionally, instead of a trivial averaging of weights learned across edge devices (as is done in the current federated learning setup), we provide a way for weighted averaging of these weights, done in a periodic manner at a logic layer residing on a server. This allows for learned weightage of a given node in decision-making for that node. This could be a system (administration) level input too, where people accountable for a certain use-case decide to give higher weightage to the output of one node over another. We provide a mathematical formulation for these methods and an algorithmic framework for their deployment in the real world.

 

As an extension to accommodate complex Spatio-temporal scenarios  – a Cross-Node Federated Graph Neural Network was investigated– which explicitly encodes the underlying graph structure using graph neural network (GNN)-based architecture under the constraint of cross-node federated learning, which requires that data in a network of nodes is generated locally on each node and remains decentralized. This graph model operates by disentangling the temporal dynamics modeling on devices and spatial dynamics on the server, utilizing alternating optimization to reduce the communication cost, facilitating computations on the edge devices.
