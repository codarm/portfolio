OVERVIEW

With the recent advancement in Urban transit and traffic sensing, data storage, and communication technologies, the availability and diversity of transportation data have increased substantially. Integrating multi-source Urban Transit data to conduct analysis is becoming a challenging task. The heterogeneous spatiotemporal resolutions of such data and the lack of standard geospatial representations of multi-source data are the main hurdles for solving the data-integration problem for multi-modal transit.

The current Patent filed work proposes a transportation data-integration framework based on a uniform geospatial roadway/city referencing layer. The efforts of building the framework are on the basis of traffic sensors’ locations and sensing areas, transportation-related data which can be classified into four categories, including on-road segment-based data, off-road segment-based data (Segments), on-road point-based data, and off-road point-based data (Nodes). A data-integration solution is proposed accordingly. An iterative map conflation map-matching algorithm as a core component of the framework is proposed for integrating the on-road segment-based data. The overall integration performance and the efficiency of the iterative map conflation algorithm in terms of the percentage of integrated roadway segments and computation time are analyzed from a real-time pipeline perspective. To produce efficient transportation analytics, the proposed framework is implemented on an interactive data-driven transportation analytics platform.

 

The data integration refers to the integration of multiple spatiotemporal transportation data sets. The data-integration process contains two aspects: spatial integration and temporal integration. A set of n spatiotemporal transportation data sets that need to be integrated can be denoted by X = {X1; : : : ; Xn } . The Spatio-temporal values in each data set are represented by Xi ∈ RTi×Di   in which Ti and Di are the temporal and spatial dimensions, respectively, of the ith data set. In addition, each data set Xi is associated with a geospatial/geometric representation of its corresponding roadway network, denoted by Gi. A uniform referencing layer covering the traffic/city networks of all the data sets is denoted by G0. Then, given X, G = {G1; :::;Gn} , and G0, the geospatial integration process can be defined as a function F such that

F({(X1, G1}; : : : ; (Xn, Gn); G0) =  ({X1, G1, L1), : : : , (Xn, Gn, Ln)}; G0 )

 

Where Li has generated linkage information that builds a link between Gi and G0. The core of the spatial integration is to find the best linkage data set L =  {L1, : : : ,Ln}. As for the temporal integration process, although transportation data sets may have different temporal resolutions, the data integration can be easily carried out using the minimum temporal resolutions among all data sets as the temporal resolution of the integrated data. Then the temporal resolution of the integrated data can be aggregated to a proper scale for further analysis if needed. Thus, the key problem of the transportation data integration is the geospatial data- integration process. (Refer to Block Diagram Fig 1)

 

Optimal Transport Idea for Map Matching

 

In defining one of the possible Linkage Layers, we have employed Optimal Transport ideas. Traditional matching methods use buffering and geometrical distances for potential matches searching, and for this, the coordinate systems need to be similar. These methods may be invalid if the coordinate systems are unknown. Therefore, a novel approach of comparing the structures of urban road networks based on network graph extraction is proposed in the patent-pending framework. In this context of matching data from two different heterogenous map sources defined via their network graphs,  solving for the graph matching is important due to its applications to define a normalized layer for data across sources.  However, the linear assignment problem is a computational bottleneck which leads to inefficient matching though the accuracy is good. To balance the efficiency and accuracy, we have leveraged optimal transport ideas.

 

Replacing the linear sum assignment step in graph matching with the "Optimal Transport" method of Cuturi (2013)  we were able to achieve improvements and solve the mobility data conundrum in a time bound fashion which can be scaled for dynamic data source
